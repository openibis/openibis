OpenIBIS

AI Operating System

Open architecture for real-world agent-driven operations

View on GitHub
🦅

8 Specialized Agents

Henry (Ops), Sofia (Client Comms), Marcus (Marketing), Vera (Compliance), Lane (Finance), Felix (Recruiting), Aria (Knowledge), KAIROS (Background Tasks)

🧠

Memory Palace

Persistent knowledge organized in browsable rooms. AutoMemory detects emerging themes and creates new rooms overnight.

🔄

Self-Evolving

Nightly AutoDream cycle: consolidates memory, extracts patterns, compresses history, generates new capabilities.

📱

Telegram-Native

Runs entirely from Telegram. Morning briefings, task management, SMS relay, domain management — all from chat.

🤖

Multi-Model Routing

Routes each request to the optimal AI model via OpenRouter. Fast queries to GPT-4o-mini, reasoning to Claude, long tasks to Gemini.

🔐

3-Layer Resilience

Git auto-backup every 6 hours, nightly encrypted rsync, AES-256 snapshots. Boots from cold backup in under 2 minutes.

Architecture

Single Node.js process. SQLite memory. Deploy anywhere.

ibis.js (core) ---> router.js (multi-model) ---> OpenRouter API | | v v SQLite Telegram Bot API Claude / GPT / Gemini | v MemPalace (knowledge rooms) | v Agents: henry, sofia, aria, kairos, autodream...

Quick Start

# Clone and configure git clone https://github.com/irinafain30-bit/openibis.git cd openibis cp .env.example .env # Add your API keys npm install # Run node ibis.js # Production (auto-restart + boot persistence) pm2 start ibis.js --name ibis pm2 save && pm2 startup
